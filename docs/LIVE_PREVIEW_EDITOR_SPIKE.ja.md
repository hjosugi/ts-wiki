# ライブプレビューエディタスパイク

ステータス: オプションのエディタモードとして実用可能；Markdownやビジュアル編集の置き換えはしない。

関連issue: #260。

## 目標

Markdownソースは編集可能なまま、一般的な構文をインラインでレンダリングするSilverBulletスタイルの編集モードを探る。対象ユーザーはWriteとPreviewを頻繁に切り替えずにMarkdownを書け、かつソーステキストを見て修正できる。

## 現状のエディタ基盤

- Markdownソースが正典である。
- CodeMirror 6がMarkdownエディタを支えている。
- 非エンジニア向けにビジュアルモードがあり、一般的なブロックの往復編集が可能。
- 協調的自動保存は現在Markdownテキストに依存し、レンダリングされたDOMには依存していない。

## 実現可能なアプローチ

後で3つ目のエディタモードとして `Live` を追加する。

- 別のcontenteditableレンダラーではなく、CodeMirror 6のデコレーションを使う。
- 見出し、強調、リンク、タスクリストの構文マーカーを隠すかスタイル変更する。
- コールアウト、埋め込み、イベント、画像プレビューのブロックウィジェットは、ブロックがアクティブに選択されていない時のみレンダリングする。
- キーボードナビゲーションやコピー＆ペーストでソーステキストを完全に復元可能に保つ。
- 可能な限り既存のMarkdownレンダラーをプレビューに再利用するが、編集可能なドキュメントに任意のレンダリング済みHTMLを注入するのは避ける。

## 非目標

- ビジュアルモードの置き換え。
- Markdownエディタの置き換え。
- 完全なWYSIWYGエンジンの実装。
- 最初のプロトタイプで全てのMarkdown拡張をサポートすること。

## プロトタイプの範囲

1. 見出し、太字・斜体、リンク、タスクチェックボックス、画像用の内部専用CodeMirror拡張を追加。
2. 既存のフェンス付きブロックに対して読み取り専用のブロックウィジェットを追加。
3. 大きなページでの入力遅延を測定し、モード公開前に評価。
4. プロトタイプがソース選択、元に戻す／やり直す、貼り付け、IME入力、モバイル入力を扱えるようになってからユーザープリファレンスを追加。

## 決定

後でCodeMirrorデコレーションの限定的なプロトタイプとして進める。現在のビジュアル優先のデフォルトは非エンジニア向けとして正しい道であり、ライブプレビューはパワーユーザー向けの執筆モードであって、メインの編集アーキテクチャではない。
