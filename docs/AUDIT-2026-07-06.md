# Comprehensive Audit — 2026-07-06

A full multi-angle audit of ts-wiki, conducted with parallel deep-dives into
UX/design, search, customization/flexibility, server code quality, web code
quality, missing features, plus external research into Wiki.js (Requarks/wiki)
and five alternative wikis (BookStack, Outline, Docmost, Trilium, SilverBullet).

**Outcome: 218 issues filed (#82–#299)**, organized into the tracks below.

## Product direction set during this audit

- **Concept: `kawaii-wiki.ts`** (#261 rebrand) — shift the design language from
  corporate-docs minimalism toward pop/game-wiki/kawaii culture (lit.link as a
  customization reference), while making the product radically usable for
  non-engineers. Design vision RFC: #262.
- **VTuber-wiki enablement** — first-class embeds (YouTube/Twitch/X/pixiv/BOOTH),
  stream-schedule bridges to the existing event system, profile infoboxes, and a
  starter template set (#284–#293).
- **Frontend direction RFC (#294)** — Lean Vue (recommended) vs Quasar (#295–#296)
  vs Nuxt (#297–#298) vs headless-UI hybrid (#299). Research note: none of the
  surveyed wiki-class OSS uses Nuxt or Vue; Docmost is NestJS+React/Mantine,
  Outline is Koa+React. The SEO goals driving Nuxt interest are mostly covered
  by cheap targeted issues (bot meta injection #201, sitemap #200).

## Suggested sequencing (dependency-aware)

1. **Foundations**: CSS-variable layer (#114) → functional dark mode (#113) →
   shared Modal (#172) + toasts (#165) + useAsyncData (#239); app.ts split (#209);
   renderer plugin seam (#123); markdown.ts split (#234).
2. **Security batch**: #82, #83, #84, #85, #87 (small, high value).
3. **Concept wave**: rebrand (#261), design RFC (#262) → theme presets (#263) and
   the kawaii/VTuber blocks, which all sit on the foundations above.
4. **Framework RFC (#294)** decided by the two spikes before any large UI rewrite.
